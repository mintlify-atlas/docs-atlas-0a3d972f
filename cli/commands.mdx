The EnergiaPro CLI provides two main commands for interacting with the API: `installations` and `measurements`.

## Global Options

These options are available for all commands:

| Option | Short | Environment Variable | Description | Default |
|--------|-------|---------------------|-------------|--------|
| `--username` | `-u` | `ENERGIAPRO_USERNAME` | EnergiaPro API username | Required |
| `--secret-key` | `-k` | `ENERGIAPRO_SECRET_KEY` | EnergiaPro API secret key | Required |
| `--format` | `-f` | - | Output format (text, csv, json, jsonl, parquet) | `text` |
| `--timeout-secs` | - | - | HTTP timeout in seconds | `30` |
| `--base-url` | - | `ENERGIAPRO_BASE_URL` | Custom API base URL | - |

## installations

Fetch and display all installations for a specific client.

### Usage

```bash
energiapro installations [OPTIONS] <CLIENT_ID>
```

### Arguments

<ParamField path="CLIENT_ID" type="string" required>
  The EnergiaPro client identifier
</ParamField>

### Options

All [global options](#global-options) are available.

### Output Fields

When using structured output formats (csv, json, jsonl, parquet), the following fields are included:

- `id` - Installation identifier
- `street_name` - Street name
- `street_address` - Full street address
- `building_number` - Building number
- `postal_code` - Postal code
- `city` - City name

### Examples

<CodeGroup>
```bash Default (text table)
energiapro installations CLIENT_123
```

```bash CSV output
energiapro installations CLIENT_123 --format csv
```

```bash JSON output
energiapro installations CLIENT_123 -f json
```

```bash Save to file
energiapro installations CLIENT_123 -f parquet > installations.parquet
```
</CodeGroup>

## measurements

Fetch and display measurements for a specific installation.

### Usage

```bash
energiapro measurements [OPTIONS] <CLIENT_ID> <INSTALLATION_ID>
```

### Arguments

<ParamField path="CLIENT_ID" type="string" required>
  The EnergiaPro client identifier
</ParamField>

<ParamField path="INSTALLATION_ID" type="string" required>
  The installation identifier (num_inst)
</ParamField>

### Options

| Option | Short | Description | Default |
|--------|-------|-------------|--------|
| `--scope` | `-s` | Scope to query (lpn-json, gc-plus-json) | `lpn-json` |
| `--from` | - | Start date filter in YYYY-MM-DD format | - |
| `--to` | - | End date filter in YYYY-MM-DD format | - |

All [global options](#global-options) are also available.

### Output Fields

- `client_id` - Client identifier (integer)
- `installation_id` - Installation identifier
- `timestamp` - Measurement timestamp
- `index_m3` - Meter index in cubic meters
- `consumption_m3` - Consumption in cubic meters
- `consumption_kwh` - Consumption in kilowatt-hours

### Examples

<CodeGroup>
```bash All measurements
energiapro measurements CLIENT_123 INST_001
```

```bash Date range filter
energiapro measurements CLIENT_123 INST_001 \
  --from 2024-01-01 \
  --to 2024-01-31
```

```bash Different scope
energiapro measurements CLIENT_123 INST_001 --scope gc-plus-json
```

```bash JSON output
energiapro measurements CLIENT_123 INST_001 \
  --from 2024-01-01 \
  --format json
```
</CodeGroup>
